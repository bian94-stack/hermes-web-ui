# WSL Auto-Start Deployment Guide

一键实现 Hermes Web UI 的**隐形自启**——开机即用，无需手动打开终端窗口。

## 功能特性

| 特性 | 说明 |
|------|------|
| **静默运行** | 无黑色终端窗口弹出 |
| **开机即用** | 重启后直接访问 `localhost:8648` |
| **关闭终端不受影响** | WSL 内核持续保活 |
| **崩溃自愈** | systemd 自动重启服务 |
| **脱离 Docker** | 纯 Node.js 进程，内存占用极低 |

## 系统要求

- Windows 10/11 with WSL2
- Ubuntu (或其他 WSL 发行版)
- Node.js v18+
- systemd 已启用 (WSL2 默认支持)

## 安装步骤

### 第一步：WSL 内配置（systemd 服务）

在 WSL 终端中执行：

```bash
# 进入项目目录
cd ~/hermes-web-ui

# 构建生产版本（如未构建）
npm run build

# 运行安装脚本
bash deployment/wsl-auto-start/install.sh
```

安装脚本会：
1. 创建 `~/.config/systemd/user/hermes-ui.service`
2. 建立静态文件软链接（修复 ENOENT 错误）
3. 启用 linger（后台驻留）
4. 启动服务

### 第二步：Windows 端配置（计划任务）

在 **Windows PowerShell（管理员）** 中执行：

```powershell
# 进入项目目录（Windows 路径）
cd D:\Users\Z\hermes-web-ui\deployment\wsl-auto-start

# 运行安装脚本
.\install.ps1 -WSLUser z -WSLDistro Ubuntu
```

安装脚本会：
1. 创建计划任务 `HermesAutoStart`（SYSTEM 级，开机触发）
2. 写入 VBS 保活脚本到 Startup 文件夹

### 第三步：测试

1. **重启电脑**
2. **不要打开任何 WSL 终端**
3. 等待 30-60 秒
4. 浏览器访问 `http://localhost:8648`

如果能正常对话，说明配置成功！

## 文件说明

| 文件 | 位置 | 作用 |
|------|------|------|
| `hermes-ui.service` | `~/.config/systemd/user/` | systemd 用户级服务 |
| `keep_wsl_alive.vbs` | Windows Startup 文件夹 | 隐藏窗口保活 WSL |
| `HermesAutoStart` 任务 | Windows 计划任务 | 开机自动唤醒 |

## 常见问题

### Q: 关闭终端后 Web 无法访问？

检查保活机制是否生效：
```bash
# WSL 内检查服务状态
systemctl --user status hermes-ui.service

# Windows 检查计划任务
schtasks /query /tn "HermesAutoStart"
```

### Q: 重启后 Web 无法访问？

等待更长时间（WSL 内核初始化需要时间），然后检查：
```powershell
# Windows 检查计划任务是否运行
Get-ScheduledTask -TaskName "HermesAutoStart" | Get-ScheduledTaskInfo
```

### Q: 服务崩溃怎么办？

systemd 配置了 `Restart=always`，崩溃后 5 秒自动重启。如需手动重启：
```bash
systemctl --user restart hermes-ui.service
```

### Q: 如何完全禁用自动启动？

```bash
# WSL 内禁用服务
systemctl --user stop hermes-ui.service
systemctl --user disable hermes-ui.service

# Windows 删除计划任务
schtasks /delete /tn "HermesAutoStart" /f

# 删除 VBS 脚本
del "%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\keep_wsl_alive.vbs"
```

## 内存占用

实际进程内存占用约 **250MB**：
- Node.js (Web UI): ~90MB
- Hermes Gateway (Python): ~140MB

Windows 任务管理器显示的 `vmmemWSL` 约 **1-2GB**，包含虚拟机 overhead 和缓存，属于正常范围。

## 卸载

```bash
# WSL 内
systemctl --user stop hermes-ui.service
systemctl --user disable hermes-ui.service
rm ~/.config/systemd/user/hermes-ui.service
sudo rm /var/lib/systemd/linger/$(whoami)
```

```powershell
# Windows PowerShell (Admin)
schtasks /delete /tn "HermesAutoStart" /f
del "%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\keep_wsl_alive.vbs"
```
